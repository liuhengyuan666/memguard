# Skill 能力拓展 ROI 分析

> 分析时间：2026-05-09  
> 分析目标：在 OpenCode + OhMyOpenAgent + Memguard 组合下，评估短期可编码能力的 ROI，找到 token 消耗与收益的最佳平衡点。  
> 用户场景：个人 vibe-coding，token-budget 敏感

---

## 评估方法

### ROI 公式

```
ROI = (激活频率 × 痛点解决度 × 10) / (预期 Token 成本/千 + 复杂度)
```

- **激活频率**：0-10，该能力在典型会话中被触发的概率
- **痛点解决度**：0-5，该能力解决实际问题的程度
- **Token 成本/千**：单次会话中该能力增加的 token 数（除以 1000）
- **复杂度**：0-5，实现和维护该能力的工程复杂度
- **ROI > 10**：极高价值，应立即实施
- **ROI 5-10**：高价值，建议实施
- **ROI 2-5**：中等价值，可延迟或按需实施
- **ROI < 2**：低价值，跳过或暂缓

## 候选能力评估

### A. ADR 扩展 decisions.md

| 维度 | 评分 | 说明 |
|------|------|------|
| Token 成本/激活 | ~250-350 | 模板字段增加 + 规则加载 |
| 预期 Token/会话 | ~100-140 | 条件加载，非每次必载 |
| 激活频率 | 4/10 | 每次涉及决策时触发 |
| 痛点解决度 | 4/5 | 直接防止决策循环讨论 |
| 复杂度 | 1/5 | 仅需扩展模板和 1 条规则 |
| **ROI** | **~14.3** | **排名第一** |

**收益分析**：
- 避免一次重复决策讨论：节省 ~500-2000 token
- 每月假设遇到 2 次重复决策：节省 ~1000-4000 token
- 每月增加成本：~3000-4200 token（30 会话 × 100-140）
- **净收益：中性到显著正收益**（取决于项目复杂度）

**实施成本**：5 分钟，3 个文件修改

## 关键警告

### Token 累积效应

如果同时加载 A+B+C+D+E+F，每次会话会增加 **~3.5k-5k token**，这对个人项目完全不值得。

**最佳设计原则**：
- 每个能力保持在 **500-800 token** 以下
- **按触发条件加载**，而非默认加载
- 先跑通 A，验证 ROI，再考虑 D 和 C
